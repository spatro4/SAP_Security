# SAP Security Architect Academy

**Master SAP Security & Authorization from Consultant to Principal Architect.**

A production-ready, fully static interview-preparation platform for experienced SAP Security & Authorization consultants (12+ years) targeting Senior Consultant, Lead, Architect, Manager, and Principal Architect level interviews. Everything is markdown/JSON-driven, statically exported, and deployable to GitHub Pages with zero backend.

## Features

- 15 deep learning tracks (ECC, S/4HANA, Fiori, Gateway, ABAP Authorization, Authorization Troubleshooting, SAP BTP, IAS, IPS, SSO/SAML/Kerberos/X.509, OAuth & Principal Propagation, HANA Security, Network/Transport Security, GRC/SoD/Audit, Production Support & Migration), each following an 18-section deep-dive template (Beginner → Principal Architect depth, internal working, real production examples, troubleshooting, audit perspective, architecture, decision-making, FAQs).
- A large, structured interview question bank (hints, expected + detailed answers, follow-ups, common mistakes, tcodes/tables/authorization objects, real scenarios, architecture diagrams) across Easy → Principal Architect difficulty.
- Scenario Labs: real production-support-style troubleshooting write-ups.
- Quiz Engine (MCQ, True/False, Scenario, Drag-Drop), Flash Cards with spaced repetition, and a timed Mock Interview simulator with self-scoring and weak-area feedback.
- Architecture diagram gallery (Mermaid), Authorization Object / Transaction / Table Explorer, Glossary, Cheat Sheets, and 30/90/365-day revision roadmaps.
- Dashboard with learning progress, readiness score, weak areas, bookmarks, daily challenge, and revision-due tracking — all persisted locally via `localStorage` (no backend, no accounts).
- Dark/Light theme, fully responsive, installable PWA (offline shell caching), sitewide search (Fuse.js), and print-friendly / PDF-exportable pages (browser print → Save as PDF).

## Tech Stack

Next.js (App Router, static export) · React · TypeScript · Tailwind CSS · shadcn/ui-style components (Radix primitives) · Framer Motion · Mermaid.js · Prism (via `rehype-prism-plus`) · Zustand · Fuse.js · GitHub Actions → GitHub Pages.

## Getting Started

```bash
npm install

```

## Building

```bash
npm run build      # runs the search-index prebuild step, then next build (static export to ./out)
```

The production build is a fully static export in `./out` — you can serve it with any static file server (`npx serve out`) to verify the final output before deploying.

## Deploying to GitHub Pages

This repo ships a ready-to-use GitHub Actions workflow (`.github/workflows/deploy.yml`) that builds and deploys to GitHub Pages automatically on every push to `main`.

**One-time setup:**

1. Push this repository to GitHub (already done if you're reading this from the repo).
2. In the repository, go to **Settings → Pages**, and under **Build and deployment → Source**, select **GitHub Actions**.
3. Push to `main` (or run the workflow manually from the **Actions** tab) — the site will build and publish to `https://<your-org-or-user>.github.io/SAP_Security/`.

**Base path**: `next.config.mjs` automatically sets `basePath`/`assetPrefix` to `/SAP_Security` only when running inside GitHub Actions (`GITHUB_ACTIONS=true`), so local `npm run dev`/`npm run build` still work at the root path. If you rename the repository, update the `repoName` constant at the top of `next.config.mjs` to match.

## Adding Content

The site is designed to grow purely by adding files — no code changes required for new content.

- **New chapter**: add a Markdown file to `content/chapters/`, with frontmatter matching `ChapterFrontmatter` in `lib/content-types.ts` (`title`, `category` — must match a slug in `lib/categories.ts` — `slug`, `order`, `summary`, `tags`, `estMinutes`, `level`). Use the existing 18-section heading structure (Beginner Concepts → FAQs) as your template; Mermaid code fences (```mermaid) and syntax-highlighted code blocks are both supported automatically.
- **New interview questions**: add or extend a JSON array in `content/questions/<category>.json` matching the `InterviewQuestion` interface.
- **New scenario**: same pattern in `content/scenarios/<category>.json` matching `Scenario`.
- **New flashcards / quiz questions / glossary terms / architecture diagrams / explorer entries**: add to the corresponding JSON file(s) under `content/flashcards`, `content/quiz`, `content/glossary`, `content/architecture`, `content/explorer/*`.
- **New cheat sheet**: add a Markdown file to `content/cheatsheets/` with `title` and `summary` frontmatter.
- **New category**: add an entry to `CATEGORIES` in `lib/categories.ts` (this automatically adds it to navigation, chapter/question/scenario category pages, and the dashboard).

After adding content, `npm run build` regenerates the search index and validates every page still builds statically — run it before committing.

## Local Data & Privacy

All personal state (progress, bookmarks, notes, quiz/mock-interview history, spaced-repetition schedule) is stored entirely in the browser's `localStorage` — there is no backend, no account system, and no data leaves the device. Use **Settings → Export Backup** to download a JSON snapshot, and **Import Backup** to restore it (e.g., when switching browsers/devices).
